# O-Rider vs Strava 웹 기능 비교

> 최종 업데이트: 2026-02-14

## 비교 요약

O-Rider 웹 대시보드는 Strava 웹의 핵심 기능을 대부분 구현하였으며, 일부 고급 기능은 미구현 상태.

| 영역 | 구현도 | 비고 |
|------|--------|------|
| 활동 피드 | 90% | 피드, 좋아요, 댓글, 검색 구현. 클럽 피드 미구현 |
| 활동 상세 | 85% | 지도, 차트, 세그먼트, 사진 구현. Matched Runs/Laps 미구현 |
| 프로필 | 80% | 통계, 트렌드, 팔로우 구현. 트로피/배지 미구현 |
| 소셜 | 75% | 팔로우, 알림, 댓글, 좋아요 구현. 클럽/챌린지 미구현 |
| 세그먼트 | 70% | 리더보드, KOM/QOM 구현. Segment Explore/별 표시 미구현 |
| 라우트/지도 | 40% | 경로 표시만. 라우트 빌더 미구현 |
| 구독 기능 | N/A | O-Rider는 무료 서비스 |

## O-Rider만의 차별점

1. **한국 특화**: 전체 한국어 UI, 한국 자전거 커뮤니티 링크 (도싸, 자출사 등)
2. **무료**: 모든 기능 무료, 광고 없음
3. **앱 연동**: KMP 모바일 앱과 실시간 데이터 동기화 (Firestore)
4. **자동 백업**: 3시간마다 Firestore GCS 백업
5. **Strava 마이그레이션**: 큐 기반 대량 임포트 + Rate Limit 관리
6. **동행 자동 매칭**: 30% 시간 겹침 기반 그룹 라이드 자동 감지

## 주요 미구현 기능 (Strava 대비)

| 우선순위 | 기능 | 난이도 | 비고 |
|----------|------|--------|------|
| 높음 | 라우트 빌더 | 높음 | Leaflet Draw 또는 OSRM 연동 필요 |
| 높음 | 클럽/그룹 | 중간 | GroupDashboard 플레이스홀더 이미 존재 |
| 중간 | 개인 히트맵 | 중간 | 활동 GPS 데이터 집계 + Leaflet.heat |
| 중간 | 장비 관리 | 낮음 | Firestore 컬렉션 추가 |
| 중간 | Laps/Splits | 중간 | 스트림 데이터에서 구간 분할 |
| 낮음 | 챌린지 | 높음 | 전체 시스템 설계 필요 |
| 낮음 | 연간 목표 | 낮음 | 프로필에 목표 필드 추가 |
| 낮음 | 트로피/배지 | 중간 | 달성 조건 정의 + 알림 |
